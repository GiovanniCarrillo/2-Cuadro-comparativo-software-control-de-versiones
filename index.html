Conclusión:

Git, Apache Subversion (SVN) y Monotone son tres sistemas de control de versiones de sofware libre, sin embargo, algunos son mas populares que otros, en el caso de Monotone es un sistema distribuido creado en C++ que sirvió de inspiración a Linus Torvalds para crear Git y por las fechas de sus últimas actualizaciones podemos decir que Git llegó a remplazar a Monotone, Git se presentó como un potente sistema de control de versiones distribuido el cual ha sido bien adoptado hoy en día y por su reciente actualización podemos decir que ha sido bien recibido por los usuarios que requieran control de versiones. En el caso de Apache Subversion es un Sistema de control de versiones el cual es más lento en comparación a Git debido a su enfoque centralizado.

Estas son buenas herramientas para el control de versiones, y el uso de cada una depende de las necesidades del usuario y a pesar de tener diferencias, comparten un objetivo en común, ayudar al usuario a tener una buena administración con las versiones de sus proyectos, lo cual es muy útil para organizar flujos de trabajo y prevenir conflictos en la colaboración.
